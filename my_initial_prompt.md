Role:
Act as a software architect and designer of children's educational experiences.

Objective:
Help me design an interactive educational web application aimed at a 10-year-old child, focused on learning world flags and countries through gameplay.

Context:
My child shows a strong interest in world flags. He enjoys learning:
- Country names
- Their flags and colors
- The description and meaning of the flags
- Capitals
- Continents
- Interesting and cultural facts about each country

He currently uses a physical card game with flags, and I would like to bring that experience into a more interactive and scalable web application.

Functional Requirements:
1. A complete catalog of countries that includes:
   - Flag image
   - Country name
   - Capital
   - Continent
   - Flag description
   - Country facts (cultural, geographical, or interesting trivia)

2. An interactive world map that:
   - Displays continents differentiated by colors
   - Includes political divisions of countries
   - Assigns a number to each country
   - Allows visually linking a flag number to its corresponding country on the map

3. Game system:
   - Start with a simple initial game: “Guess which country this flag belongs to”
   - Design the architecture to be scalable in order to add more games and menus in the future

Design Requirements:
- Visually appealing design for children
- Simple, clear, and elegant style
- Highly intuitive and interactive interface
- Playful and educational approach

Expectations:
- Propose an overall application structure
- Suggest appropriate web technologies (frontend and, if applicable, backend)
- Describe the flow of the first game
- Provide UX/UI recommendations for children
- Design the solution in a modular and scalable way
